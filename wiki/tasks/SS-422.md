# Structured display of warnings vs critical issues

## Summary

Present validation findings in a structured form that separates non-critical
recommendations/warnings from critical issues that prevent the skill(s) from passing.

## Unit test requirements

- Given fixed props with mixed findings, critical issues render in a distinct blocking
  group and non-critical recommendations render in a separate warning group, each with
  an actionable description
- Given props with zero findings, the component renders a "passed with no issues" state
  and no empty severity groups
- Given props containing at least one critical finding, the computed overall
  pass/fail state resolves to not-passing regardless of how many recommendations are
  present, and the critical group is visually emphasized

## Integration test requirements

- Render the component with a mocked API client returning a completed validation with
  mixed findings; assert the critical and recommendation groups render distinctly from
  the fetched data
- Render the component with a mocked API client returning a clean-pass result; assert
  the "passed with no issues" state is shown and no empty groups render
- Render the component with a mocked API client returning a result containing a
  critical issue; assert the overall status displayed is not-passing and the critical
  group is emphasized
