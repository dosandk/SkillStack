# Profile page with GitHub information

## Summary

Give a logged-in user a profile page that displays their GitHub information (such as
username, avatar, and profile details).

## Unit test requirements

- Profile component renders GitHub username, avatar, and profile details correctly given
  a populated user object
- Profile component renders a graceful error/placeholder state when GitHub info is
  unavailable, without throwing
- Profile component renders a loading state while GitHub info is being fetched

## Integration test requirements

- Rendering the profile page with a mocked auth/API client returns and displays the
  signed-in user's GitHub information, matching the account used to sign in
- Rendering the profile page when the GitHub info fetch fails (mocked client error)
  shows the graceful placeholder and the page does not crash
- Navigating to the profile route while unauthenticated redirects to sign-in via the
  session guard, and no profile data is fetched or exposed
