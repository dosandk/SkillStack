# Validate button triggers on-demand validation

## Summary

Give the content owner a Validate button on their repository or individual skill that
triggers on-demand validation, which always fetches the latest version from GitHub.

## Unit test requirements

- Validate button renders enabled for the owner and is not rendered/disabled for a
  non-owner, given fixed ownership props
- Clicking Validate transitions local component state to an in-progress state
  immediately, before any response is received
- On a service error/timeout response, the component's error-handling logic surfaces a
  failure message and leaves the previously displayed status untouched
- Retry logic re-triggers the validation action and resets the error state

## Integration test requirements

- Render the component with a mocked API client; clicking Validate calls the mocked
  on-demand validation endpoint and the UI shows the in-progress state until the mock
  resolves
- Mock the API client to reject/time out; assert the UI displays the failure message,
  keeps the prior status, and allows a retry that re-invokes the mocked endpoint
- Render the component as a non-owner with a mocked API client; assert the Validate
  action is unavailable and no call is made to the endpoint
