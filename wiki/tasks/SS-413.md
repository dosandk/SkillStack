# Restrict upload to authenticated users

## Summary

Enforce that only authenticated users can reach and use the upload UI, so anonymous
visitors are prompted to sign in rather than shown a usable upload form. Server-side
enforcement of unauthenticated/expired-session requests is handled by the Firestore
security rules delivered in SS-504, not reimplemented here.

## Unit test requirements

- Route-guard decision logic returns "allow" for an authenticated auth state and "deny"
  for an unauthenticated auth state
- Route-guard decision logic treats an expired/missing session the same as unauthenticated

## Integration test requirements

- Rendering the upload route with a mocked authenticated auth state shows the usable
  upload form
- Rendering the upload route with a mocked unauthenticated (or expired-session) auth
  state hides/disables the upload form and redirects to or prompts sign-in, against a
  real or mocked auth state provider
