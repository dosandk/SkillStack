# Login flow + protected route guard

## Summary

Wire a login/logout control (e.g. in the root layout's nav) to `lib/auth.tsx`, and add a
protected-route guard so `/profile` (SS-203) redirects a signed-out visitor to the
catalog/login state instead of rendering.

## Unit test requirements

- The login control calls the auth context's login function on click; the logout
  control calls logout.
- The route guard redirects when there is no current user, and renders its child route
  when there is one.

## Integration test requirements

- Navigating to `/profile` while signed out redirects; navigating to it while signed in
  (mocked auth state) renders the profile route.
