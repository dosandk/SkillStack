# Free-text search + tag/agent filtering

## Summary

Let users narrow the catalog by a free-text query and by tag/agent filters, combining
multiple filters to refine results. An earlier version of this filtering logic was built
against the now-deleted legacy `db/content` mock data — treat that implementation as
reference only, since it needs to be rebuilt against the new Firestore/Cloud-Functions-
backed data shape, not reused as-is.

## Unit test requirements

- Filter logic narrows entries by a free-text query matching against title/description
- Filter logic narrows entries by one or more selected tag/agent filters
- Query and filters are AND-ed: only entries matching the query and all selected filters remain, entries matching just one criterion are excluded
- Clearing all filters restores the full, unfiltered list
- A query/filter combination matching nothing produces an empty result set

## Integration test requirements

- Render the search + filter UI with a mocked list of entries, type a query and select tag/agent filters, and assert the rendered list narrows to matching entries, a "no results" state appears for a non-matching combination, and clearing filters recovers the full list
