# Repository lookup function (commit hash + statuses)

## Summary

A Cloud Function that, given a repository reference, checks Firestore and returns whether
the repository exists along with its stored commit hash and the validation status of each
skill (pending/approved/failed). Used by the CLI before install.

## Unit test requirements

- Request-validation logic: given a call payload, determines whether the repository
  reference is present and well-formed before any Firestore query is attempted.
- Response-shaping logic: given a Firestore document (or its absence), builds the
  exists/commit-hash/per-skill-status response shape returned to callers.

## Integration test requirements

- Seed the Firestore emulator with a repository document and invoke the function —
  assert it returns `exists=true` with the stored commit hash and per-skill validation
  statuses matching the seeded data.
- Invoke the function against the emulator for a repository reference with no matching
  document — assert it returns `exists=false` with no fabricated data.
- Invoke the function with a missing or malformed repository reference — assert it
  returns a client error and the emulator is not queried with an invalid reference.
