# Commit-diff comparison + choice prompt (validated + changed)

## Summary

When `scanRepository` (SS-601's contract) reports a known repo whose `commitHash`
differs from what the CLI last installed, and the repo's stored status is `validated`:
tell the user the repo has changed and let them choose which version to install — the
stored/validated one, or the new latest one ([FR-CLI6](../requirements.md#fr-cli6)).

## Unit test requirements

- Given `known: true`, a differing commit hash, and `repoValidationStatus: "validated"`,
  the command presents the choice prompt.
- Choosing "stored" installs against the previously-recorded commit; choosing "latest"
  installs against the new one.
- Given a matching commit hash (no change), no prompt appears — install proceeds as
  normal.

## Integration test requirements

- Running the command against a mocked `scanRepository` response for each branch above
  (changed+validated choosing stored, changed+validated choosing latest, unchanged)
  reaches the correct install path.
