# Best-practice checks & structured result output

## Summary

Extend validation to also evaluate non-critical best practices, and produce a structured
result that separates critical issues from non-critical recommendations/warnings for
storage and UI display.

## Unit test requirements

- Prompt construction includes best-practice checks alongside the existing
  security/convention checks, given fixed input fixtures
- Findings-categorization logic classifies a given raw finding as a non-critical
  best-practice recommendation, distinct from critical findings
- Schema validation of the LLM's structured output enforces the agreed shape
  (severity, category, message) and handles unknown/empty categories consistently
- Overall status computation reflects only critical findings, given a fixed set of
  mixed critical and non-critical findings

## Integration test requirements

- Call the validation function against a mocked Anthropic SDK response containing both
  critical and non-critical findings, with the Firestore emulator; assert the written
  result has distinct critical/recommendation groups and the overall status reflects
  only the critical findings
- Call the validation function against a mocked Anthropic SDK response containing only
  best-practice findings, with the Firestore emulator; assert the skill is written as
  approved with the recommendations persisted and available for display
- Call the validation function against a mocked Anthropic SDK response and assert the
  persisted result conforms to the structured schema consumed by the UI
