# Auth context over Firebase Auth state

## Summary

`client/src/lib/firebase.ts` already sets up `auth` and `githubProvider` (plus the auth
emulator in dev) — the login mechanism exists but nothing in the app reads it yet
([FR-UI3](../requirements.md#fr-ui3)). Add
`client/src/lib/auth.tsx`: a thin React Context wrapping Firebase Auth's
`onAuthStateChanged`, exposing the current user (or `null`) and login/logout functions.
Not a state-management library — plumbing only, per [AD-4](../architecture-invariants.md#ad-4).

## Unit test requirements

- The context provider exposes `null` before Firebase reports an auth state.
- The context provider exposes the signed-in user's GitHub profile fields once
  `onAuthStateChanged` fires with a user.
- Calling the exposed login function invokes `signInWithPopup` (or redirect) with
  `githubProvider`; calling logout invokes `signOut`.

## Integration test requirements

- A component tree wrapped in the provider, rendered against a faked/mocked Firebase
  Auth instance, reflects a login → logged-in → logout cycle correctly.
