# Skill listing as cards

## Summary

Render the catalog of skills as a grid of cards on the landing page, each showing the
key metadata a browsing user needs to decide whether to open a skill. An earlier version
of this card grid was built against the now-deleted legacy `db/content` mock data — treat
that implementation as reference for markup/interaction patterns only, since it needs to
be rebuilt against the new Firestore/Cloud-Functions-backed data shape, not reused as-is.

## Unit test requirements

- Card component renders title, description, and key metadata for a single entry given fixed props
- Grid renders one card per entry for a populated list, laid out responsively
- Empty-entry list renders an empty-state message instead of a blank grid, with no error thrown
- An entry missing required fields is skipped (or rendered degraded) without crashing the render of the remaining entries

## Integration test requirements

- Render the card grid with a mocked data source returning a mixed list (valid entries plus one malformed entry) and assert only the valid cards render, the malformed entry is skipped or degraded, and the page does not crash
