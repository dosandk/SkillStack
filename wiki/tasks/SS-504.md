# Telemetry Cloud Function

## Summary

A Cloud Function the CLI calls after a successful install
([FR-CLI9](../requirements.md#fr-cli9)): increments the installed skill's own install
count, then calls `repositories-store.ts`'s `recalculateInstallCount(repoId)` ([AD-11](../architecture-invariants.md#ad-11)) to
roll that up onto the repository doc — this function touches install count only, never
validation status. It also marks the installed repo/skill(s) as awaiting validation
(status `pending` if not already `validated`/`failed`). No auth required — this is
anonymous telemetry, not an owner-scoped write. Reused as-is by story-cli-install-update
(SS-603).

## Unit test requirements

- Given a repo/skill identifier, the function increments that skill's own install count
  and calls `recalculateInstallCount` so the repository-level total reflects it.
- Calling it for a repo/skill that's already `validated` does not reset it to
  `pending` — only a genuinely new/changed install does that.
- This function never writes a skill's or repository's `validationStatus` field —
  only `installCount` — per [AD-11](../architecture-invariants.md#ad-11)'s split (aggregation helpers are one-field-at-a-time,
  called only by the writer that actually changed that field).

## Integration test requirements

- Calling the function against the Firestore emulator increments the expected install
  counts (skill-level and repository-level rollup) and sets the expected pending status,
  while leaving `validationStatus` untouched.
