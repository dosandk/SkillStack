# Query repositories owned by the authenticated user

## Summary

Implement a Cloud Function/query that, given the authenticated caller's uid, returns
every repository document in Firestore owned by that uid along with each one's
calculated validation status. This is what SS-432 (frontend) renders on the profile
page; the query is scoped strictly to the caller's own uid and must never expose
another user's repository data.

## Unit test requirements

- Query filter correctly constrains results to documents whose owner field equals the
  given uid
- Validation status calculation correctly derives each repository's current status from
  its underlying validation records
- Zero-results case (caller owns no repositories) returns an empty array without error
- Invocation without a valid authenticated uid (missing/invalid auth context) is
  rejected before any Firestore read is attempted

## Integration test requirements

- Calling the function against the Firestore emulator seeded with repositories from
  multiple owners returns only the documents owned by the caller's uid, each with the
  correct calculated validation status
- Calling the function while unauthenticated (no valid auth context) is rejected and no
  repository data is returned
- Calling the function for a caller who owns zero repositories returns an empty result
  set against the seeded emulator data
- Calling the function for a caller whose repositories have a mix of validation states
  returns statuses matching the seeded validation records exactly
