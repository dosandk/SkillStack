# React Router v8 setup + catalog/search page

## Summary

Install and configure React Router v8 in Data Mode (`createBrowserRouter`) — not yet a
dependency of `client/`. Build the root layout and the first route,
`client/src/routes/catalog/`, the public search page (no login required, per
[FR-UI1](../architecture.md#fr-ui1)/[FR-UI2](../architecture.md#fr-ui2)). Its `loader` calls `client/src/lib/api.ts` (new — the one HTTP client to
`functions/`, per AD-4); no component calls `fetch()` directly.

## Unit test requirements

- The catalog page renders a list of results from a given loader payload.
- Searching filters the rendered list by name (skill or repository).
- An empty result set renders a clear empty state, not a blank page.

## Integration test requirements

- The route's `loader` calls `lib/api.ts`'s search function with the expected request
  shape and renders its response.
