# GitHub API enrichment (stars & live metadata)

## Summary

Provide a backend path that queries the GitHub API to fetch up-to-date repository
metadata (such as stars) and merges it with stored Firestore data for display in the UI.

## Unit test requirements

- Merge function combines a stored Firestore repository record with a GitHub API metadata response into one object, with stored validation/commit fields taking precedence over any conflicting live fields
- Merge function omits or marks live fields as stale when the GitHub metadata input is null/errored, while stored fields remain unchanged
- Cache/throttle helper determines whether a cached metadata value is still fresh vs. needs refetching, given a last-fetched timestamp and a TTL

## Integration test requirements

- Call the enrichment function against the Firestore emulator for a known repository with a mocked GitHub API, and assert the merged result includes stars/live fields alongside stored data
- Call the enrichment function with the mocked GitHub API returning an error/rate-limit response, and assert stored data is still returned with live fields omitted/marked stale and no hard failure
- Call the enrichment function repeatedly within a short window against a mocked GitHub API, and assert the GitHub API is not called on every request (cache/throttle respected) while stored validation/commit fields remain authoritative
