# Skills subcollection schema

## Summary

Define the `skills` subcollection under each repository document. Each skill carries a
name, validation status (pending/approved/failed), installation count from telemetry,
and created/updated timestamps.

## Unit test requirements

- The skill document schema accepts a payload with name, validation status
  (pending/approved/failed), install count, and timestamps
- The skill document schema rejects a status value outside
  pending/approved/failed
- Install count validation/clamping logic rejects or clamps a negative value per the
  documented rule

## Integration test requirements

- Adding a skill to a repository's `skills` subcollection against the Firestore
  emulator persists the document linked to its parent repository
- Attempting to write a skill with a status outside pending/approved/failed is
  rejected against the Firestore emulator and the skill is not persisted with an
  invalid status
- Attempting to set or overwrite the install count from an untrusted client is
  rejected against the Firestore emulator, while an increment via the telemetry
  function succeeds and the stored count remains consistent
