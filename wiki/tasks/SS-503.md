# Calculated repo validation status & install count

## Summary

Compute and store the repository-level validation status and total installation count as
derived fields aggregated from the `skills` subcollection, keeping them consistent as
skills change.

## Unit test requirements

- The aggregation function computes repository validation status and total install
  count from an array of skills with differing statuses and install counts, matching
  the documented aggregation rules
- The aggregation function given an empty skills array returns the defined default
  validation status and zero install count, with no divide-by-zero or undefined state

## Integration test requirements

- Running the aggregation Cloud Function against the Firestore emulator for a
  repository with several skills stores repo-level aggregate values that match the
  underlying skills
- Changing a skill's status or install count in the Firestore emulator triggers
  recomputation, and the repository-level aggregates are updated and do not go stale
- Running the aggregation Cloud Function against a repository with no skills in the
  Firestore emulator stores the defined default status and zero install count
