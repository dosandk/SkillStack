# Direct-to-GitHub content download for selected skill(s)

## Summary

Per AD-7: once a skill path is known (from SS-501's scan result), fetch that path's
actual file content directly from the GitHub API — extending the existing
`download-repo.ts` blob-fetch, but scoped to the known skill path(s) rather than a
blind repo-wide tree walk. Keeps file-transfer bandwidth on the user's own GitHub
connection instead of funneling through `functions/`.

## Unit test requirements

- Given a skill path, the fetch returns every file under that path (any sub-depth),
  matching AD-6's "whole directory" rule.
- A fetch failure for one file surfaces a clear error and does not silently produce a
  partial skill directory.

## Integration test requirements

- Fetching against a stubbed GitHub Blobs API for a known multi-file skill path
  reconstructs the full directory content correctly (text and binary files both).
