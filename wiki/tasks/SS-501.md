# `add` command calls `scanRepository`; unvalidated-repo warning; skill selection

## Summary

Wire the CLI's `add <repo-url> [--skill <name>]` command to call the backend's
`scanRepository` (SS-301) instead of doing its own discovery — per AD-6, discovery
lives only in `functions/`. If the response says the repo is unknown, warn the user it's
unvalidated and ask for confirmation before proceeding ([FR-CLI5](../architecture.md#fr-cli5)). If `--skill` wasn't
given, list every skill the response discovered and let the user pick one or more
([FR-CLI4](../architecture.md#fr-cli4)).

## Unit test requirements

- Given a scan response for an unknown repo, the command prints the unvalidated warning
  and halts until the user confirms.
- Given `--skill` omitted, the command presents every discovered skill for selection and
  accepts multiple selections.
- Given `--skill <name>` matching a discovered skill, the command proceeds directly
  without a selection prompt.
- Given `--skill <name>` matching nothing discovered, the command reports no match and
  exits without installing anything.

## Integration test requirements

- Running the command against a mocked `scanRepository` HTTP response end to end
  reaches the correct next step (warning / selection prompt / direct proceed) for each
  case above.
