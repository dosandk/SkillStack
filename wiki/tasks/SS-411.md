# URL upload input & submission

## Summary

Provide a logged-in user with a URL input field to submit a GitHub repository or skill
for inclusion in the registry. On submission, the form validates the URL client-side and
calls the create-entry Cloud Function (SS-412), then reflects the resulting pending state
back to the user.

## Unit test requirements

- Valid GitHub repository/skill URL passes client-side validation
- Malformed or non-GitHub URL is rejected by client-side validation with an inline error
- Submit handler is disabled/no-ops while a submission is already in flight

## Integration test requirements

- Submitting a valid URL calls the (mocked) create-entry endpoint with the expected
  payload and renders a "pending" confirmation on success
- Submitting an invalid URL does not call the create-entry endpoint and shows the inline
  validation error instead
- A create-entry call that returns an error (e.g. already exists) surfaces that message
  to the user without leaving the form in a stuck "submitting" state
