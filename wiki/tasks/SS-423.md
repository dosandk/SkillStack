# Reflect updated validation status in UI

## Summary

After validation completes, update the entry's displayed status (approved/failed) in the
UI and in listings so users see the current result.

## Unit test requirements

- Status-badge mapping logic returns the correct badge/label for each status value
  (pending, approved, failed), given a fixed status prop
- Repo-level status aggregation is a pure function of its skills' statuses and matches
  the documented aggregation rule for representative mixes (all approved, all failed,
  mixed, all pending)
- Given a previously rendered pending status and a new status value in props, the
  component re-renders the new status rather than retaining the stale one

## Integration test requirements

- Render the detail view and a listing with a mocked API client returning an updated
  status after validation; assert both views show the new status badge consistently
- Render with a mocked API client returning a status change from pending to failed;
  assert the failed status is shown on revisit rather than the stale pending value
- Render a repository view with a mocked API client returning skills with mixed
  statuses; assert the displayed repo-level status matches the aggregation rule and is
  consistent with the per-skill statuses shown
