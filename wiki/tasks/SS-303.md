# Upload UI

## Summary

Build `client/src/routes/upload/` (protected — logged-in only): a URL input field that
calls the upload endpoint (SS-302) via `lib/api.ts`, and displays the resulting
repository with its skill(s) in `pending` status once the call succeeds.

## Unit test requirements

- Submitting a valid GitHub URL calls the upload API with that URL and the current
  user's auth token.
- A successful response renders the repository and its skill(s) as pending.
- An error response (e.g. invalid URL, scan failure) renders a clear error, not a blank
  state.

## Integration test requirements

- The full submit → API call → pending-state-render path works against a mocked
  `lib/api.ts` response.
