# Capture full skill directory contents

## Summary

Once a `SKILL.md` is located, capture the entire containing skill directory including
all subdirectories regardless of their nesting depth, so nothing the skill needs is
left behind.

## Unit test requirements

- Path-scoping logic: given a skill directory root and a list of repo paths, correctly
  determines which entries belong under the skill directory (recursively) versus
  sibling/parent paths, as a pure filter with no real filesystem/API access.
- Failure-handling logic: given a simulated failed fetch for one file within a skill,
  decides to abort that skill's capture entirely rather than proceed with the rest.

## Integration test requirements

- Capture a skill from a fixture repo whose directory contains nested subfolders and
  assets — assert every file/subdirectory under the skill directory is included and
  the relative structure is preserved.
- Simulate one file failing to fetch mid-capture (via a fixture/mocked API) — assert
  the CLI reports the specific failure, aborts that skill's capture, and does not
  produce a partial, silently incomplete skill.
- Capture a skill from a fixture repo that also has sibling directories outside the
  skill folder — assert only content under the skill's own directory is included and
  sibling/parent content is excluded.
