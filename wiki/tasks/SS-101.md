# GitHub OAuth login flow

## Summary

Allow a user to sign in with their GitHub account via OAuth so the app can establish an
authenticated identity used for uploads, validation, and profile access.

## Unit test requirements

- OAuth callback handling parses a successful auth response and derives the local
  session/user identity data correctly
- OAuth callback handling recognizes a denied/cancelled consent response (error param
  present) and does not construct a session
- Redirect-target resolution correctly identifies the sign-in path for a protected route
  when no session is present

## Integration test requirements

- Clicking "Sign in with GitHub" and completing the OAuth consent screen against a real
  auth provider redirect results in an authenticated session with the GitHub identity
  available to the app
- Cancelling/denying the GitHub OAuth consent screen returns the user to the app still
  unauthenticated with a non-blocking message, and no session is created
- Attempting to open an authenticated-only route (e.g. upload) while unauthenticated
  triggers a redirect to sign-in and the protected action is not performed
