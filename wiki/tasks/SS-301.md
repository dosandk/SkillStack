# Repository scan for SKILL.md up to depth 3 & list discovered skills

## Summary

Using the GitHub API, scan a target repository for skills defined by a `SKILL.md` file
located at most three levels deep (root, `/skills/<name>/`, `/skills/<group>/<name>/`),
ignoring anything nested deeper. When the user runs the add command without a `--skill`
argument, present all discovered skills and let them select one or more to install; when
`--skill <name>` is given, match it against the discovered list. (Merged with the former
SS-302 — discovery and the resulting skill-selection prompt are one cohesive scan step.)

## Unit test requirements

- Depth calculation: given a list of repo file paths, correctly classifies each
  `SKILL.md` as depth 1/2/3 or deeper, independent of any real GitHub call.
- Repo URL parsing/validation logic flags a malformed or empty URL before any API call
  is attempted.
- GitHub API error classification logic maps rate-limit and auth error responses to the
  correct internal error type used to drive user-facing messaging.
- Multi-select state logic: given a list of discovered skill names and selection input,
  produces the correct set of chosen skills as a pure in-memory computation.
- "No skills found" logic: given an empty discovery result, produces the correct
  report/exit decision without invoking any prompt.
- Explicit `--skill <name>` matching logic: given a discovered skill list and a name
  that does not match, returns a not-found result listing available skills, as a pure
  lookup with no IO.

## Integration test requirements

- Run the scan against a fixture repo with `SKILL.md` files at depths 1, 2, and 3 —
  assert all three are discovered and each is identified by its containing directory.
- Run the scan against a fixture repo with a `SKILL.md` nested at depth 4 — assert it
  is excluded from the results.
- Run the scan against a nonexistent/invalid repository URL — assert a clear
  "repository not found/accessible" error and a non-zero exit code.
- Run the scan against a fixture/mocked GitHub API response simulating a rate-limit or
  auth error — assert a clear surfaced message and that no partial skill list is
  reported as complete.
- Run the CLI against a fixture repo with multiple discovered skills and no `--skill`
  flag — assert every discovered skill is listed in a multi-select prompt and the
  selection is carried forward to installation.
- Run the CLI against a fixture repo with zero discovered skills — assert it reports
  none found and exits without prompting for selection.
- Run the CLI against a fixture repo with discovered skills and a `--skill <name>` that
  matches none of them — assert it reports the skill was not found, lists available
  skills, and installs nothing.
