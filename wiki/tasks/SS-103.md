# GitHub metadata enrichment for search

## Summary

Extend the list/search response to blend each repository's stored Firestore fields with
a live GitHub API call for display metadata (stars, current default branch), per [FR-BE3](../requirements.md#fr-be3).
Owned by a dedicated `functions/src/services/github-client.ts` — separate from the CLI's
own GitHub client ([AD-6](../architecture-invariants.md#ad-6)/[AD-7](../architecture-invariants.md#ad-7) split: this one serves metadata, never file content).

## Unit test requirements

- The enrichment function merges one stored repository record with a mocked GitHub API
  response into a single response object with both stored and live fields.
- A GitHub API failure for one repository doesn't fail the whole list — that repo's live
  fields are omitted/null while the rest of the list still returns.

## Integration test requirements

- Calling the list endpoint against the Firestore emulator with a stubbed GitHub API
  returns stored + live fields merged for every repository in the result.
