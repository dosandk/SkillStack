# Auto-install-latest path (not yet validated + changed) + telemetry re-fire

## Summary

When `scanRepository` reports a known repo whose commit hash differs but the stored
status is **not** `validated` (still `pending` or `failed`): install the latest version
directly, same as the first-time flow, with no choice prompt ([FR-CLI7](../architecture.md#fr-cli7)). After install,
call the telemetry function (SS-504, reused as-is) to record the install and mark the
repo/skill(s) pending again.

## Unit test requirements

- Given `known: true`, a differing commit hash, and `repoValidationStatus` of `pending`
  or `failed`, the command installs the latest version without prompting.
- After a successful install via this path, the telemetry call fires with the new
  commit hash.

## Integration test requirements

- Running the command against a mocked `scanRepository` response for this branch
  installs the latest content and triggers the telemetry call exactly once.
