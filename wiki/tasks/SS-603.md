# On-demand validation for owners

## Summary

Expose validation as an on-demand operation that a logged-in owner can trigger for their
own skill or entire repository, fetching the latest GitHub version each time.

## Unit test requirements

- Authorization logic returns allowed for the requesting user matching the
  repository/skill owner and denied otherwise, given fixed identity/ownership fixtures
- Failure-result construction logic produces a non-approved, source-not-found result
  when the GitHub fetch step reports the source is missing, without invoking the LLM
  check

## Integration test requirements

- Call the on-demand validation Cloud Function against the Firestore emulator with a
  mocked Anthropic SDK response, authenticated as the owner; assert the result is
  stored and the updated status is returned
- Call the function with a mocked GitHub fetch that reports the source no longer
  exists; assert against the Firestore emulator that the entry is written as failed
  and never marked approved
- Call the function authenticated as a non-owner; assert the request is rejected and
  the Firestore emulator shows no validation was run and no document changes occurred
