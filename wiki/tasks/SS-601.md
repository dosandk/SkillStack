# LLM validation core (security + conventions)

## Summary

Implement the core validation that uses an LLM (e.g. Anthropic SDK) to check a skill's
latest GitHub version for security issues and adherence to documented conventions,
producing a critical/pass verdict.

## Unit test requirements

- Prompt construction correctly embeds the fetched skill content and the
  security/convention rules to check, given fixed input fixtures
- Findings-categorization logic classifies a given raw finding as critical (security or
  convention violation) versus non-critical, in isolation from the LLM call
- Schema validation of the LLM's structured output accepts well-formed responses and
  rejects/flags malformed or missing fields
- Overall verdict computation returns approved only when no critical findings are
  present, given a fixed set of categorized findings

## Integration test requirements

- Call the validation function against a mocked Anthropic SDK response representing a
  clean skill, with the Firestore emulator; assert the skill is written as approved
  with no critical issues and the validated commit hash is persisted
- Call the validation function against a mocked Anthropic SDK response containing a
  security/convention violation, with the Firestore emulator; assert the skill is
  written as failed with the critical issue captured
- Seed the Firestore emulator with a stale commit hash and mock the GitHub fetch to
  return a newer HEAD; assert the function evaluates the latest version and persists
  the commit hash it actually validated
