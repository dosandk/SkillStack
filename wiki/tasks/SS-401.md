# Search & view validated skills

## Summary

Let any user (including unauthenticated visitors) search for and view individual skills
that have passed validation, excluding pending or failed entries, by wiring the catalog
UI to the public search endpoint (SS-514).

## Unit test requirements

- Response-to-view-model mapping transforms the search endpoint's skill payload into the shape the card/detail components expect
- An empty results array maps to a "no results" view state without requiring extra flags
- Guard logic never treats a non-validated skill payload (status not validated) as viewable/public content

## Integration test requirements

- Render the search view with a mocked API client returning validated skills matching a keyword, and assert only validated results appear and each can be opened to a detail view
- Render the search view with a mocked API client returning a query that only matches non-validated skills, and assert the "no results" state is shown and no pending/failed skill data reaches the DOM
- Render the skill detail view with a mocked API client for a direct link to a non-validated skill id, and assert it is not shown as public/validated content
