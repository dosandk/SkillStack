# Owner-scoped upload endpoint

## Summary

Replace the current `apiWriteRepository`/`write-repository.ts` (accepts any body, no
auth check, old minimal schema) with an upload endpoint that: verifies the caller's
Firebase Auth ID token via the admin SDK (AD-9); calls `scanRepository` (SS-301) for the
skill list; and creates the `repositories` doc (owner = verified caller) plus one
`skills` subcollection doc per discovered skill, each with status `pending` and an empty
`findings` array (SS-101's schema).

## Unit test requirements

- A request with a missing/invalid ID token is rejected before any Firestore write.
- A request with a valid token and a scan result creates one repository doc (owner =
  token's uid) and one skill doc per discovered skill, all `pending`.
- Re-uploading an already-known repo (per SS-301's existence check) updates rather than
  duplicates the existing docs.

## Integration test requirements

- Calling the endpoint against the Firestore emulator with a valid mocked ID token and
  a stubbed scan result persists the expected repository + skills documents.
- Calling it with an invalid token persists nothing.
