# Search repositories

## Summary

Allow users to search for entire repositories (not just individual skills) and view a
repository with its list of validated skills.

## Unit test requirements

- Repository search response mapping groups results by repository and attaches each repo's validated skill list
- Repository entries whose validated skill list is empty are filtered out of the mapped view models before render
- Result-list mapping distinguishes repository result items from skill result items within the same result set

## Integration test requirements

- Render the repository search view with a mocked API client returning repositories with validated skills, and assert each repo lists its validated skills and links to the correct detail view
- Render the repository search view with a mocked API client returning a repository whose skills are all pending/failed, and assert it is not surfaced as a result
- Render a combined search results view with a mocked API client returning both a matching repository and a matching skill, and assert the two result types render as distinct entries linking to their respective detail views
