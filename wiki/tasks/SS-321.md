# Firestore lookup call + unvalidated-repo warning

## Summary

Before installing, the CLI calls a Cloud Function that checks Firestore for the target
repository. If the repository is not in the database, the CLI warns the user that it is
unvalidated so they can decide whether to proceed at their own discretion.

## Unit test requirements

- Response-handling logic: given a mocked lookup response shape (exists/commit
  hash/statuses), decides how to proceed (use the info vs. warn) without any real
  network call.
- Warning/confirmation decision logic: given an unvalidated-repo result and a
  simulated user confirm/decline input, computes whether to proceed or abort.
- Error classification logic distinguishes a "service unreachable" error from other
  error shapes to select the correct user-facing message path.

## Integration test requirements

- Call the actual repository-lookup Cloud Function against the Firestore emulator for
  a repo document seeded in Firestore — assert the CLI receives the stored commit hash
  and per-skill validation statuses and proceeds using that data.
- Call the lookup function against the emulator for a repository with no matching
  document — assert the CLI displays the unvalidated-repository warning and prompts
  for confirmation, and that declining aborts with no install proceeding.
- Point the CLI at an unreachable/erroring function endpoint — assert it surfaces a
  clear message, requires explicit confirmation before proceeding, and never silently
  treats the repo as validated.
