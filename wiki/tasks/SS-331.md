# Post-install telemetry request

## Summary

After a successful installation, the CLI calls a Cloud Function to record that the
repository was used and the installation completed, so installation counts can be
aggregated.

## Unit test requirements

- Send-decision logic: given an install run's final state (nothing installed vs.
  something installed), decides whether telemetry should be sent at all, as a pure
  function over an install-result object.
- Payload-construction logic: given the set of installed skill(s)/repo, builds the
  correct telemetry request body.
- Non-blocking-failure logic: given a failed telemetry call outcome, determines that
  install success reporting is unaffected and a non-fatal warning is produced.

## Integration test requirements

- After a successful install, call the actual telemetry Cloud Function against the
  Firestore emulator — assert the function records the installation identifiable by
  repository and installed skill(s).
- Point the telemetry call at an unreachable/erroring function endpoint after a
  successful install — assert the CLI still reports install success to the user and
  surfaces the telemetry failure as a non-fatal warning.
- Run an install that is aborted/cancelled before anything is installed — assert no
  call is made to the telemetry function and the emulator shows no new or incremented
  record.
