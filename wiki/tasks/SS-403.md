# Validate UI

## Summary

On the owner's own repository/skill view (reached from the profile page, SS-203, or the
upload result, SS-303), add a "Validate" button that calls SS-402's on-demand endpoint,
and render the result: overall status plus critical issues and non-critical
recommendations as two visually distinct, structured sections (never one paragraph of
text — per [FR-UI7](../architecture.md#fr-ui7)).

## Unit test requirements

- Clicking "Validate" calls the on-demand endpoint for the current repo/skill.
- A response with critical findings renders them in a distinct "critical" section and
  shows an overall failed/blocked state.
- A response with only recommendations renders them in a distinct "recommendations"
  section and shows an overall validated state.

## Integration test requirements

- The full click → API call → structured-results-render path works against a mocked
  `lib/api.ts` response with both severities present.
