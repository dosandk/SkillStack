# Deploy client and Cloud Functions

## Summary

Set up deployment for the React + Vite client and the Firebase Cloud Functions back end
so that both are reachable in a hosted environment and the front end can call the
functions endpoint.

## Unit test requirements

- Deploy config validation logic rejects a config object missing required Firebase
  project fields before the deploy pipeline is invoked
- Error-response formatting logic for a function invoked without required environment
  configuration produces a safe payload (500 status, generic message, no secret
  values) given a mocked missing-config state

## Integration test requirements

- Running the deploy pipeline against a clean checkout with valid Firebase
  configuration deploys the client and functions; the client is served from its
  hosting URL, the functions respond to health checks, and the client is configured
  to call the deployed functions base URL
- Running the deploy pipeline against a missing or invalid Firebase configuration
  aborts with a descriptive error before publishing, leaving the previously deployed
  version live and unchanged
- Invoking a deployed function without required environment configuration returns a
  500 with a safe error message containing no secret values, and the failure is
  logged for diagnostics
