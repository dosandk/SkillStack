# Public search endpoint for validated content

## Summary

A Cloud Function search endpoint that returns only validated skills and repositories for
the public UI, combining stored data with the enrichment path (SS-513) where needed.

## Unit test requirements

- Query-matching logic filters a set of skills/repositories to only those with validated status matching the search term
- Query-matching logic returns an empty result set for a query with no validated matches, without throwing
- Result-filtering logic excludes pending/failed entries even when they match the search term more strongly than validated entries

## Integration test requirements

- Call the search Cloud Function against the Firestore emulator seeded with validated and non-validated entries, and assert only validated skills/repositories matching the query are returned
- Call the search Cloud Function against the Firestore emulator with a query that matches nothing validated, and assert an empty result set is returned without error
- Call the search Cloud Function as an unauthenticated caller against the Firestore emulator seeded with validated and non-validated entries sharing matching terms, and assert no non-validated content is present in the response
