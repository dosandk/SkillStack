# Sort tabs (Trending/Hot/All Time) + status badges

## Summary

Provide time-windowed sort tabs for the catalog and show a status/audit badge on each
card so users can gauge popularity and trust at a glance. An earlier version of this
sorting/badge logic was built against the now-deleted legacy `db/content` mock data —
treat that implementation as reference only, since it needs to be rebuilt against the
new Firestore/Cloud-Functions-backed data shape, not reused as-is.

## Unit test requirements

- Sort logic orders entries by installs desc for "All Time"
- Sort logic orders entries by recency desc for "Hot"
- Sort logic orders entries by recency-weighted installs for "Trending"
- An entry lacking install count or creation date is ordered deterministically at the end rather than throwing
- Status badge component maps each known audit status to the correct color/label
- An unknown/unrecognized status renders no misleading badge

## Integration test requirements

- Render the tabbed list with a mocked dataset that mixes complete entries, entries missing sort fields, and entries with varied/unknown audit statuses; switch sort tabs and assert the list re-sorts correctly and each card's badge renders consistently with its status
