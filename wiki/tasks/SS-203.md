# Profile page

## Summary

Build `client/src/routes/profile/` — a protected route (guarded by SS-202) that
displays the logged-in user's GitHub info (avatar, username/display name, GitHub
profile link) sourced from the Firebase Auth user object exposed by `lib/auth.tsx`. No
backend call needed — GitHub identity data comes straight from the Auth provider.

## Unit test requirements

- The profile page renders the current user's avatar, name, and GitHub link from a
  given auth-context user object.
- Rendering with no current user (e.g. mid-redirect) doesn't throw.

## Integration test requirements

- The route, rendered inside the guard (SS-202) with a mocked signed-in user, displays
  that user's GitHub info end to end.
