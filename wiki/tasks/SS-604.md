# Daily scheduled validation of unvalidated skills

## Summary

Run automatic validation once per day for skills that are still unvalidated, fetching the
latest GitHub version and updating their status and findings.

## Unit test requirements

- Selection logic filters a fixed list of skills down to only those still unvalidated
  (e.g. `pending`), excluding already-approved/failed skills unless explicitly due
- Per-item error-isolation logic catches a single item's thrown/rejected validation,
  logs it, and allows batch processing to continue with the remaining items
- Run-lock logic prevents a second scheduled run from starting while one is already
  in progress, given a fixed "already running" state

## Integration test requirements

- Seed the Firestore emulator with skills in mixed statuses (pending, approved, failed)
  and a mocked Anthropic SDK response; run the scheduled job and assert only the
  still-unvalidated skills are validated and updated, while approved/failed skills are
  left untouched
- Seed the Firestore emulator with multiple pending skills and mock the Anthropic SDK
  to throw for one skill; run the job and assert the remaining skills are still
  validated, the failure is logged, and the batch completes
- Simulate an in-progress run against the Firestore emulator and trigger the scheduled
  job again; assert the overlapping run is prevented and only one run's writes are
  persisted
