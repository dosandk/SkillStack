# Multi-select target platform prompt & install skill directory into platform folders

## Summary

After skills are selected, prompt the user to choose one or more target platforms
(Claude, Cursor, Copilot) that the skill(s) should be installed for, then copy each
selected skill's captured directory into the correct platform-specific folder in the
user's local project for every chosen target platform. (Merged with the former SS-312 —
the platform choice and the install step that consumes it are one cohesive unit.)

## Unit test requirements

- Platform selection state logic: pure state transitions for toggling/recording chosen
  platforms from multi-select input.
- Empty-selection validation logic: given no platforms selected, decides between
  re-prompting and aborting with the required-platform message.
- Non-interactive `--platforms` flag parsing/validation logic: normalizes a valid
  subset of platform values and rejects invalid platform names, independent of any
  actual prompt or install step.
- Destination-path resolution logic: given a platform and skill name, computes the
  correct platform-specific target folder path as a pure function with no filesystem
  access.
- Existing-skill conflict decision logic: given a simulated disk state and a force-flag
  value, decides whether to warn, overwrite, or skip, without touching real files.

## Integration test requirements

- Run the install flow selecting skills then choosing Claude and Cursor at the
  platform prompt (against a temp project directory) — assert both are recorded as
  install targets and installation proceeds for each.
- Confirm the platform prompt with no platform selected — assert the CLI re-prompts or
  aborts with the required message and nothing is written to disk.
- Run the command non-interactively with a valid `--platforms` flag — assert those
  platforms are used without prompting; run it with an invalid platform value — assert
  a clear error and no install attempted.
- Install a captured skill directory for Claude and Cursor into a real temp project
  directory — assert it is written to each platform's conventional folder with the
  nested structure preserved intact.
- Simulate a filesystem error partway through writing a skill (e.g. an unwritable temp
  path) — assert the CLI reports the failure, leaves no half-written skill for that
  platform, and accurately reports other successfully written skills/platforms.
- Install against a temp project directory where the target folder already contains
  the skill — assert the user is warned and asked to overwrite or skip, and that no
  overwrite happens without confirmation or an explicit force flag.
