# Monorepo scaffold + TypeScript path aliases

## Summary

Restructure the repository into a single-package monorepo (`client`, `functions`,
`shared`, `wiki`) plus a standalone `cli` package, and wire cross-folder imports
through TypeScript path aliases (`@shared`, `@db`, `@eleks-ui/*`) so both the front end
and Cloud Functions share a single source of truth.

## Unit test requirements

- tsconfig path-alias entries for `@shared`, `@db`, and `@eleks-ui/*` resolve to the
  expected relative paths in the base config
- Vite's `resolve.alias` config mirrors the same set of aliases defined in the tsconfig
  paths

## Integration test requirements

- Running `npm run typecheck` against a fixture module that imports a type from
  `@shared` in both the client and a function passes with no unresolved-path errors,
  and the compiled output resolves the alias to the `shared/` source
- Running `npm run typecheck` against a fixture module that imports from a folder not
  exposed via an alias fails with a clear module-resolution error and emits no partial
  build artifact
- Building the standalone `cli` package in isolation (`cd cli && npm run build`)
  succeeds without the root package installed, and the package does not import from
  `client/` or `functions/` internals
