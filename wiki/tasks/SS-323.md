# Version choice for validated repos with changes

## Summary

When a validated stored version differs from the latest commit (as determined by
SS-322), prompt the user to choose which version to install: the validated stored
version or the updated latest version. Handle the user's choice, including
cancellation, without silently carrying over the stored version's validated status to
the new commit.

## Unit test requirements

- Decision logic resolves "install stored version" when the user selects the validated
  stored version, and does not mark it as awaiting a new validation pass.
- Decision logic resolves "install latest version" when the user selects the updated
  version, and marks it as unvalidated/awaiting a new validation pass.
- Decision logic resolves "install nothing" when the user cancels the prompt, with no
  installation state produced.
- The previously-approved status is never attached to the new, differing commit hash
  regardless of which choice is made.

## Integration test requirements

- Run the CLI against a fixture repo with a validated stored version and a differing
  latest commit; select the stored version and assert the CLI installs it and that no
  telemetry marking a new version as awaiting validation is triggered.
- Run the CLI against the same fixture setup; select the updated latest version and
  assert the CLI installs it, clearly marks it as unvalidated in its output, and
  triggers telemetry recording a new installation awaiting future validation.
- Run the CLI against the same fixture setup and cancel the prompt; assert nothing is
  installed and the command exits without recording an installation.
