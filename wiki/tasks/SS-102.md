# Client-side auth state & session guard

## Summary

Maintain authenticated state on the client and guard authenticated-only routes/actions,
exposing the current user to components and clearing state on sign-out.

## Unit test requirements

- Restoring persisted auth state on init correctly reconstructs the current user object
  from stored session data
- Sign-out handler synchronously clears all local auth state (user, tokens)
- Session guard logic denies access when auth state is null/expired and permits it when
  the session is valid

## Integration test requirements

- Reloading the app with a valid persisted session restores authenticated state without
  re-prompting for login, and the current user's GitHub info is available to components
- Triggering a request that requires auth with an expired session token is treated as
  unauthenticated, clears stale user state, and prompts the user to sign in again
- Signing out immediately revokes access to authenticated-only routes end-to-end and
  clears the underlying session
