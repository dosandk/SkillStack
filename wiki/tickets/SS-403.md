# Show live GitHub metadata (stars) in results

## Summary

Augment search results and detail views with up-to-date GitHub metadata (such as star
count) fetched live via the backend and merged with the stored Firestore data.

## Acceptance criteria

### AC-1: Live metadata displayed

**Given** a validated repository result
**When** it is displayed
**Then** live GitHub metadata such as star count is shown alongside stored data
**And** the values reflect the current GitHub state.

### AC-2: GitHub metadata unavailable

**Given** the GitHub API is unavailable or rate-limited
**When** results render
**Then** stored data still displays and live metadata degrades gracefully (hidden or last-known)
**And** the page does not error.

### AC-3: Boundary — stored vs live data reconciliation

**Given** stored data and live GitHub data disagree
**When** a result is displayed
**Then** validation/commit data comes from the store while volatile metadata comes from GitHub
**And** the source of each value is consistent with its type.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → live stars shown with stored data
2. **Critical negative** → GitHub outage degrades gracefully
3. **Permission boundary** → stored vs live data sourced correctly

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
- Concurrent access scenarios (if applicable)
