# Structured display of warnings vs critical issues

## Summary

Present validation findings in a structured form that separates non-critical
recommendations/warnings from critical issues that prevent the skill(s) from passing.

## Acceptance criteria

### AC-1: Findings grouped by severity

**Given** a completed validation with mixed findings
**When** results are displayed
**Then** critical blocking issues and non-critical recommendations/warnings are shown in distinct groups
**And** each finding includes an actionable description.

### AC-2: No findings / clean pass

**Given** a validation that produced no issues
**When** results are displayed
**Then** a clear "passed with no issues" state is shown
**And** no empty severity groups are rendered.

### AC-3: Boundary — critical presence blocks pass

**Given** a validation with at least one critical issue
**When** results are displayed
**Then** the overall status is shown as not passing regardless of recommendations
**And** the critical group is visually emphasized.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → findings grouped by severity
2. **Critical negative** → clean pass shows no empty groups
3. **Permission boundary** → any critical issue blocks passing status
