# Capture full skill directory contents

## Summary

Once a `SKILL.md` is located, capture the entire containing skill directory including
all subdirectories regardless of their nesting depth, so nothing the skill needs is
left behind.

## Acceptance criteria

### AC-1: Whole directory captured

**Given** a discovered skill whose directory contains nested subfolders and assets
**When** the CLI collects the skill
**Then** every file and subdirectory under the skill directory is included
**And** the relative structure is preserved.

### AC-2: Missing/unreadable file during capture

**Given** a file that cannot be fetched (deleted mid-scan or access error)
**When** the CLI collects the skill
**Then** it reports the specific failure and aborts that skill's capture
**And** does not produce a partial, silently incomplete skill.

### AC-3: Boundary — capture scoped to the skill directory

**Given** sibling directories outside the skill folder
**When** the CLI captures the skill
**Then** only content under the skill's own directory is included
**And** sibling/parent content is excluded.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → full nested directory captured
2. **Critical negative** → unreadable file aborts with clear error
3. **Permission boundary** → capture scoped to the skill directory only

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
- Concurrent access scenarios (if applicable)
