# Best-practice checks & structured result output

## Summary

Extend validation to also evaluate non-critical best practices, and produce a structured
result that separates critical issues from non-critical recommendations/warnings for
storage and UI display.

## Acceptance criteria

### AC-1: Structured result produced

**Given** a validation run
**When** it completes
**Then** it outputs a structured result with distinct critical issues and non-critical recommendations/warnings
**And** the overall status reflects only critical issues.

### AC-2: Best-practice-only findings do not block

**Given** a skill with only best-practice (non-critical) findings
**When** validation completes
**Then** the skill still passes (approved) while recommendations are recorded
**And** the recommendations are available for display.

### AC-3: Boundary — result schema is stable

**Given** a validation result is persisted
**When** the UI consumes it
**Then** it conforms to the agreed structured schema (severity, category, message)
**And** unknown or empty categories are represented consistently.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → structured result with severity groups
2. **Critical negative** → best-practice-only findings do not block pass
3. **Permission boundary** → result schema stable for consumers

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
- Concurrent access scenarios (if applicable)
