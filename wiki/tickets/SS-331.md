# Post-install telemetry request

## Summary

After a successful installation, the CLI calls a Cloud Function to record that the
repository was used and the installation completed, so installation counts can be
aggregated.

## Acceptance criteria

### AC-1: Telemetry sent after install

**Given** a successful installation of one or more skills
**When** installation completes
**Then** the CLI sends a telemetry request identifying the repository and installed skill(s)
**And** the function records the installation.

### AC-2: Telemetry failure is non-blocking

**Given** the telemetry endpoint is unreachable
**When** the CLI attempts to send telemetry after a successful install
**Then** the installation is still reported as successful to the user
**And** the telemetry failure is surfaced as a non-fatal warning.

### AC-3: Boundary — no telemetry when nothing installed

**Given** an install run that installed nothing (aborted or cancelled)
**When** the run ends
**Then** no telemetry request is sent
**And** no installation count is incremented.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → telemetry sent after successful install
2. **Critical negative** → telemetry failure is non-blocking
3. **Permission boundary** → no telemetry when nothing installed
