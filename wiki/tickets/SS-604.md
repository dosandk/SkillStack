# Daily scheduled validation of unvalidated skills

## Summary

Run automatic validation once per day for skills that are still unvalidated, fetching the
latest GitHub version and updating their status and findings.

## Acceptance criteria

### AC-1: Daily job validates pending skills

**Given** skills with `pending` status
**When** the daily scheduled job runs
**Then** each pending skill is validated against its latest GitHub version and its status/findings updated
**And** already-approved/failed skills are not needlessly re-run unless due.

### AC-2: Job resilience to per-skill failure

**Given** one skill's validation errors during the batch
**When** the daily job runs
**Then** the remaining skills are still validated and the failure is logged
**And** the job completes without aborting the whole batch.

### AC-3: Boundary — single run per day

**Given** the scheduler
**When** validation is triggered
**Then** the automatic full-scan validation runs at most once per day
**And** overlapping runs are prevented.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → daily job validates pending skills
2. **Critical negative** → per-skill failure does not abort batch
3. **Permission boundary** → runs at most once per day, no overlap
