# Skill listing as cards

## Summary

Render the catalog of skills as a grid of cards on the landing page, each showing the
key metadata a browsing user needs to decide whether to open a skill.

## Acceptance criteria

### AC-1: Catalog renders as cards

**Given** the catalog has entries
**When** a visitor opens the landing page
**Then** each entry is rendered as a card with title, description, and key metadata
**And** the cards are laid out responsively.

### AC-2: Empty catalog state

**Given** the catalog has no entries
**When** a visitor opens the landing page
**Then** an empty-state message is shown instead of a blank grid
**And** no error is surfaced.

### AC-3: Boundary — malformed entry is skipped safely

**Given** one entry with missing required fields
**When** the listing renders
**Then** valid entries still render and the malformed one is skipped or shown degraded
**And** the page does not crash.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → cards render for populated catalog
2. **Critical negative** → empty catalog shows empty state
3. **Permission boundary** → malformed entry does not break the page
