# List of user's uploaded skills/repos

## Summary

On the profile page, list the repositories/skills the user has uploaded with each
entry's current validation status, providing quick access to manage them.

## Acceptance criteria

### AC-1: Uploaded entries listed

**Given** a logged-in user who has uploaded entries
**When** they open their profile
**Then** their uploaded repositories/skills are listed with current validation status
**And** each links to its detail/validate view.

### AC-2: No uploads yet

**Given** a logged-in user with no uploads
**When** they open their profile
**Then** an empty-state prompt to upload is shown
**And** no error occurs.

### AC-3: Boundary — only own uploads listed

**Given** entries owned by other users exist
**When** the profile list renders
**Then** only the current user's uploaded entries are shown
**And** other users' entries are excluded.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → own uploads listed with status
2. **Critical negative** → empty state for no uploads
3. **Permission boundary** → only own uploads listed
