# Search & view validated skills

## Summary

Let any user (including unauthenticated visitors) search for and view individual skills
that have passed validation, excluding pending or failed entries.

## Acceptance criteria

### AC-1: Validated skill search returns results

**Given** a set of validated and non-validated skills
**When** a visitor searches by keyword
**Then** only validated skills matching the query are shown
**And** each result can be opened to view its details.

### AC-2: No validated matches

**Given** a query that matches only non-validated skills
**When** the visitor searches
**Then** a "no results" state is shown
**And** no pending/failed skills are leaked into results.

### AC-3: Boundary — unvalidated skill not directly viewable

**Given** an unauthenticated visitor with a direct link to a non-validated skill
**When** they open it
**Then** it is not shown as public/validated content
**And** access is limited to validated data only.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → validated skills returned by search
2. **Critical negative** → no-match state excludes non-validated
3. **Permission boundary** → non-validated skill not publicly viewable

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
- Concurrent access scenarios (if applicable)
- Cross-feature interactions (if this ticket's feature depends on others)
