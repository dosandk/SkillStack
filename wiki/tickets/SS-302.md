# List discovered skills when none specified

## Summary

When the user runs the add command without a `--skill` argument, present all discovered
skills and let them select one or more to install.

## Acceptance criteria

### AC-1: Interactive skill selection

**Given** a repository with multiple discovered skills and no `--skill` provided
**When** the CLI finishes scanning
**Then** it lists every discovered skill and prompts a multi-select
**And** the user's selection is carried forward to installation.

### AC-2: No skills found

**Given** a repository where no `SKILL.md` exists within depth 3
**When** the CLI scans
**Then** it reports that no skills were found
**And** exits without prompting for selection.

### AC-3: Boundary — explicit --skill not present

**Given** a `--skill <name>` that does not match any discovered skill
**When** the CLI runs
**Then** it reports the skill was not found and lists available skills
**And** does not install anything.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → multi-select list shown and selection captured
2. **Critical negative** → no skills found reported
3. **Permission boundary** → unknown `--skill` handled with guidance
