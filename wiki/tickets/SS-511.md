# Repository lookup function (commit hash + statuses)

## Summary

A Cloud Function that, given a repository reference, checks Firestore and returns whether
the repository exists along with its stored commit hash and the validation status of each
skill (pending/approved/failed). Used by the CLI before install.

## Acceptance criteria

### AC-1: Existing repository returns data

**Given** a repository present in Firestore
**When** the function is called with its reference
**Then** it returns exists=true, the stored commit hash, and per-skill validation statuses
**And** the response matches the stored data.

### AC-2: Unknown repository

**Given** a repository not in Firestore
**When** the function is called
**Then** it returns exists=false without error
**And** no data is fabricated.

### AC-3: Boundary — malformed request rejected

**Given** a call missing the repository reference or malformed
**When** the function is invoked
**Then** it returns a client error
**And** does not query with an invalid reference.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → existing repo returns hash + statuses
2. **Critical negative** → unknown repo returns exists=false
3. **Permission boundary** → malformed request rejected
