# Deploy client and Cloud Functions

## Summary

Set up deployment for the React + Vite client and the Firebase Cloud Functions back end
so that both are reachable in a hosted environment and the front end can call the
functions endpoint.

## Acceptance criteria

### AC-1: Both targets deploy successfully

**Given** a clean checkout with valid Firebase configuration
**When** the deploy pipeline runs for client and functions
**Then** the client is served from its hosting URL and the functions respond to health checks
**And** the client is configured to call the deployed functions base URL.

### AC-2: Deploy fails cleanly on bad config

**Given** a missing or invalid Firebase project configuration
**When** the deploy is attempted
**Then** the deploy aborts with a descriptive error before publishing
**And** the previously deployed version remains live and unchanged.

### AC-3: Boundary — functions require valid runtime config

**Given** a function invoked without required environment configuration
**When** it is called
**Then** it returns a 500 with a safe error message and no secret values
**And** the failure is logged for diagnostics.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → successful deploy of both targets
2. **Critical negative** → deploy aborts on invalid config without breaking live version
3. **Permission boundary** → misconfigured function fails safely

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
- Concurrent access scenarios (if applicable)
- Cross-feature interactions (if this ticket's feature depends on others)
