# Free-text search + tag/agent filtering

## Summary

Let users narrow the catalog by a free-text query and by tag/agent filters, combining
multiple filters to refine results.

## Acceptance criteria

### AC-1: Search and filters narrow the list

**Given** a populated catalog
**When** the user types a query and selects one or more tag/agent filters
**Then** the visible list is narrowed to entries matching the query and all selected filters
**And** clearing filters restores the full list.

### AC-2: No matches

**Given** a query/filter combination that matches nothing
**When** it is applied
**Then** a "no results" state is shown
**And** the user can clear filters to recover.

### AC-3: Boundary — combined filters are AND-ed

**Given** two tags selected plus a text query
**When** results are computed
**Then** only entries matching the query and both tags remain
**And** entries matching just one criterion are excluded.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → query + filters narrow results
2. **Critical negative** → no-match state shown
3. **Permission boundary** → combined filters correctly AND-ed
