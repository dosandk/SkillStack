# Sort tabs (Trending/Hot/All Time) + status badges

## Summary

Provide time-windowed sort tabs for the catalog and show a status/audit badge on each
card so users can gauge popularity and trust at a glance.

## Acceptance criteria

### AC-1: Sort tabs reorder the list

**Given** entries with install counts and creation dates
**When** the user selects All Time, Hot, or Trending
**Then** the list re-sorts by installs desc, recency desc, or recency-weighted installs respectively
**And** each card shows its status/audit badge.

### AC-2: Missing sort data

**Given** an entry lacking install count or date
**When** a sort tab is applied
**Then** the entry is ordered deterministically at the end rather than causing an error
**And** the list still renders.

### AC-3: Boundary — badge reflects exact status

**Given** entries with differing audit statuses
**When** the list renders
**Then** each badge color/label maps to the correct status value
**And** an unknown status shows no misleading badge.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → tabs re-sort correctly
2. **Critical negative** → missing sort data handled gracefully
3. **Permission boundary** → badges map to exact status

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
