# Install skill directory into platform folders

## Summary

Copy each selected skill's captured directory into the correct platform-specific folder
in the user's local project for every chosen target platform.

## Acceptance criteria

### AC-1: Skill installed to correct folders

**Given** a selected skill and target platforms Claude and Cursor
**When** installation runs
**Then** the skill directory is written to each platform's conventional folder
**And** the nested structure is preserved intact.

### AC-2: Write failure is atomic per skill

**Given** a filesystem error occurs partway through writing a skill
**When** installation runs
**Then** the CLI reports the failure and does not leave a half-written skill for that platform
**And** other successfully written skills/platforms are reported accurately.

### AC-3: Boundary — existing skill on disk

**Given** the target folder already contains the skill
**When** installation runs
**Then** the user is warned and asked to overwrite or skip
**And** no overwrite happens without confirmation (or an explicit force flag).

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → skill written to each platform folder
2. **Critical negative** → partial write does not leave corrupt state
3. **Permission boundary** → existing skill requires overwrite confirmation

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
- Concurrent access scenarios (if applicable)
- Cross-feature interactions (if this ticket's feature depends on others)
