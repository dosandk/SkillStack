# Public search endpoint for validated content

## Summary

A Cloud Function search endpoint that returns only validated skills and repositories for
the public UI, combining stored data with the enrichment path where needed.

## Acceptance criteria

### AC-1: Validated results returned

**Given** a search query
**When** the endpoint is called
**Then** it returns only validated skills/repositories matching the query
**And** pending/failed entries are excluded.

### AC-2: Empty query / no matches

**Given** a query with no validated matches
**When** the endpoint is called
**Then** it returns an empty result set
**And** does not error.

### AC-3: Boundary — non-validated content never leaked

**Given** validated and non-validated entries share matching terms
**When** the endpoint responds
**Then** only validated entries are included regardless of match strength
**And** no unauthenticated caller can retrieve non-validated content.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → validated results returned for a query
2. **Critical negative** → no matches returns empty set
3. **Permission boundary** → non-validated content never leaked

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
- Concurrent access scenarios (if applicable)
- Cross-feature interactions (if this ticket's feature depends on others)
