# Multiple skill selection in one run

## Summary

Allow the user to install several skills from the same repository in a single command
invocation, applying the chosen platforms to all selected skills.

## Acceptance criteria

### AC-1: Batch install

**Given** the user selects three skills and two platforms
**When** installation runs
**Then** all three skills are installed for both platforms
**And** a summary lists each skill/platform result.

### AC-2: Partial batch failure

**Given** one of several selected skills fails to install
**When** the batch runs
**Then** the remaining skills still install and the failed one is clearly reported
**And** the command exit code reflects a partial failure.

### AC-3: Boundary — duplicate selection deduplicated

**Given** the same skill is selected more than once
**When** the batch runs
**Then** it is installed once per platform
**And** no duplicate writes occur.

## E2E test generation guidance

The three acceptance criteria above map directly to the minimum E2E coverage:

1. **Golden path** → batch install across skills and platforms
2. **Critical negative** → partial failure reported, others succeed
3. **Permission boundary** → duplicate selection deduplicated

Additional E2E tests should be added for:

- Data edge cases mentioned in acceptance criteria
